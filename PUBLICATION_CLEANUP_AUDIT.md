This document records the audit and cleanup performed to make the repository

## 1. SECURITY INCIDENT — exposed API tokens (RESOLVED)

**Severity: critical.** Three live IBM Quantum API tokens were found hardcoded
in plaintext in `qsvm_on_wcds/REAL_QPU.ipynb`:

| Location | Label in file | Status |
|----------|---------------|--------|
| Cell 3 source (`save_account(token=...)`) | "COMPTE PERSO" | leaked |
| Cell 26 source (`save_account(token=...)`) | "COMPTE UNIVESITY" | leaked |
| Cell 26 comment block | "COMPTE CODE123" | leaked |

The tokens were committed in `c666ce4` ("Organize source files and ignore
generated artifacts") and that commit had been **pushed to the public remote**
on both `origin/main` and `origin/repo-cleanup`. Because the repository is
public, all three tokens must be considered **compromised**.

### Actions taken
1. **Revocation:** the repository owner confirmed all three tokens were
   revoked/regenerated at the IBM Quantum Platform before any cleanup.
2. **Working tree:** the `save_account(token="...")` calls were replaced with a
   safe environment-variable approach using
   `os.environ["IBM_QUANTUM_TOKEN"]` and an optional
   `os.environ.get("IBM_QUANTUM_INSTANCE", "open")`. The comment block listing
   the three raw tokens was removed.
3. **Notebook outputs:** all execution outputs of `REAL_QPU.ipynb` were cleared.
   They contained personal local paths (`C:\Users\souha\...`) and execution
   traces, none of which are needed for reproducibility.
4. **History purge:** the three literal token strings are purged from the entire
   Git history with `git filter-repo --replace-text` (replaced with
   `***REMOVED***`). See section 6 for the exact procedure and force-push step.
5. **.gitignore:** added `.env`, `*.env`, `*.token`, `*.key`, `*.pem`,
   `secrets/`, `credentials/` so credential files can never be committed again.

> NOTE: Even after history rewrite + force-push, the previously public tokens
> remain compromised forever (they may already have been scraped). The
> revocation in step 1 is what actually closes the exposure; the history purge
> removes them from the published artifact going forward.

## 6. Git history purge procedure

The literal token strings are purged from every commit:

```
# replacements.txt (one secret per line):
#   <token1>==>***REMOVED***
#   <token2>==>***REMOVED***
#   <token3>==>***REMOVED***
python -m git_filter_repo --replace-text replacements.txt --force
```

`git filter-repo` rewrites all refs (`main`, `repo-cleanup`) and removes the
`origin` remote as a safety measure; it is re-added before pushing. Because the
leaked commit exists on both public branches, **both** must be force-pushed.

Force-push commands (run only after owner confirmation):

```
git push --force origin main
git push --force origin repo-cleanup
```

## 7. Reproducibility risks

- `REAL_QPU.ipynb` now requires the user to set `IBM_QUANTUM_TOKEN` (and
  optionally `IBM_QUANTUM_INSTANCE`) in the environment before running the
  account-setup cells. This is documented in the README / reproducibility notes.
- Clearing the real-QPU notebook outputs removes a record of a specific hardware
  run; the reported quantitative results live in the tracked result CSVs, so
  scientific reproducibility is unaffected.
